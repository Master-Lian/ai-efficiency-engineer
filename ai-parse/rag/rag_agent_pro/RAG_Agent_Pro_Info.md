# RAG Agent Pro 项目文档

## 1. 项目架构

RAG Agent Pro 是一个基于 LangChain 和 LangGraph 构建的智能图书查询系统，采用模块化架构设计。

### 1.1 系统架构图

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户输入   │────>│  Agent节点  │────>│  检索工具   │────>│  相关性评估 │
└─────────────┘     └─────────────┘     └─────────────┘     └─────┬───────┘
                                                                 │
                                                                ┌┴┐
                                                                ││
                                                             ┌──┴┴──┐
                                                             │      │
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─▼─   ┌─▼─┐
│  最终回答   │<────│  生成节点   │<────│  重写查询   │<────│ 否   │ 是 │
└─────────────┘     └─────────────┘     └─────────────┘     └──────└────┘
```

### 1.2 模块说明

| 模块 | 主要职责 | 文件位置 | 核心功能 |
|------|---------|---------|----------|
| 配置管理 | 系统参数配置 | `config.py` | API密钥、路径配置、模型设置 |
| 向量数据库 | 文档向量存储与检索 | `vector_store.py` | 文档加载、分块、向量构建、检索 |
| 工具定义 | 检索功能实现 | `tools.py` | 文档检索、书籍推荐统计 |
| 工作流节点 | 核心业务逻辑 | `nodes.py` | 智能决策、相关性评估、查询重写、回答生成 |
| 工作流图 | 流程编排 | `graph.py` | 节点连接、条件判断、流程控制 |
| 主程序 | 用户交互 | `main.py` | 命令行界面、输入处理、结果展示 |

## 2. 核心功能实现

### 2.1 向量数据库

**文件**: `vector_store.py`

- **文档加载**：支持 PDF 和 DOCX 格式文件的批量加载
- **文本分块**：使用 `RecursiveCharacterTextSplitter` 进行智能分块
- **向量构建**：使用 `HuggingFaceEmbeddings` 生成文本嵌入
- **持久化存储**：使用 Chroma 向量数据库进行持久化
- **智能检索**：根据相似度排序返回相关文档

**关键特性**：
- 优先使用本地缓存的嵌入模型，避免网络依赖
- 支持自动检测向量库是否存在，不存在则自动构建
- 采用 `all-MiniLM-L6-v2` 模型，平衡性能和精度

### 2.2 检索工具

**文件**: `tools.py`

- **`retrieve_local_docs`**：基本检索工具，返回相关文档和来源书籍
- **`retrieve_books_for_recommendation`**：增强检索工具，返回更多文档用于书籍推荐统计

**特性**：
- 支持文档内容摘要（超过500字符自动截断）
- 自动统计各书籍出现频率，用于推荐排序
- 延迟加载检索器，优化启动性能

### 2.3 工作流节点

**文件**: `nodes.py`

#### 2.3.1 Agent 节点
- **功能**：智能决策，根据用户问题决定是否调用工具
- **实现**：使用 DeepSeek 模型绑定工具，生成工具调用请求

#### 2.3.2 相关性评估节点
- **功能**：评估检索结果与用户问题的相关性
- **实现**：使用 DeepSeek 模型进行相关性判断，返回 "generate" 或 "rewrite"
- **策略**：最多尝试重写2次，超过则直接生成回答

#### 2.3.3 查询重写节点
- **功能**：当检索结果不相关时，重写用户查询
- **实现**：使用 DeepSeek 模型将原始问题转换为更适合检索的形式

#### 2.3.4 生成节点
- **功能**：基于检索结果生成最终回答
- **实现**：
  1. 检索更多文档用于书籍推荐统计
  2. 基于检索到的文档生成详细回答
  3. 推荐5本最相似的书籍
  4. 整合回答和推荐结果

### 2.4 工作流图

**文件**: `graph.py`

- **构建方法**：使用 `StateGraph` 构建有向图工作流
- **节点连接**：
  - `agent` → `retrieve` (调用工具)
  - `retrieve` → `grade_documents` (评估相关性)
  - `grade_documents` → `generate` (相关) 或 `rewrite` (不相关)
  - `rewrite` → `agent` (重新查询)
  - `generate` → `END` (完成)

## 3. 技术栈

| 技术/库 | 用途 | 版本/来源 |
|---------|------|-----------|
| Python | 开发语言 | 3.10+ |

## 4. 配置说明

**文件**: `config.py`

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥 | 从环境变量加载 |
| `KNOWLEDGE_BASE` | 知识库路径 | 从环境变量加载 |
| `PERSIST_DIR` | 向量库持久化目录 | `./chroma` |
| `EMBEDDING_MODEL` | 嵌入模型名称 | `sentence-transformers/all-MiniLM-L6-v2` |
| `RETRIEVAL_K` | 检索返回文档数量 | 3 |
| `CHUNK_SIZE` | 文本分块大小 | 1000 |
| `CHUNK_OVERLAP` | 分块重叠大小 | 100 |

## 5. 使用方法

### 5.1 环境准备

1. **安装依赖**：
   ```bash
   pip install langchain langgraph chromadb langchain-huggingface langchain-openai python-dotenv
   ```

2. **配置环境变量**：
   创建 `.env` 文件：
   ```
   DEEPSEEK_API_KEY=your_deepseek_api_key
   KNOWLEDGE_BASE=path_to_your_knowledge_base
   ```

### 5.2 运行系统

**交互式模式**：
```bash
python main.py
```

**测试模式**：
```bash
python test_book_query.py
```

### 5.3 使用流程

1. 启动系统后，输入查询内容（例如："C++ 如何创建窗口"）
2. 系统会自动：
   - 分析查询意图
   - 检索相关文档
   - 评估检索结果相关性
   - 如需要，重写查询并重新检索
   - 基于检索结果生成详细回答
   - 推荐5本最相关的书籍
3. 输入 "exit" 退出系统

## 6. 性能优化

1. **模型缓存**：优先使用本地缓存的嵌入模型，避免网络下载
2. **延迟加载**：检索器采用延迟加载模式，优化启动速度
3. **批量处理**：文档加载支持多线程，提高处理速度
4. **智能重写**：通过查询重写提高检索准确性
5. **向量存储**：使用 Chroma 进行高效向量存储和检索

## 7. 扩展建议

1. **文档类型扩展**：增加对更多文档格式的支持（如 TXT、Markdown 等）
2. **多模型支持**：集成更多大语言模型选项
3. **前端界面**：开发 Web 或 GUI 界面，提升用户体验
4. **知识库管理**：增加知识库自动更新和管理功能
5. **个性化推荐**：基于用户历史查询优化推荐算法

## 8. 故障排查

### 8.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| API 密钥错误 | 环境变量未设置 | 检查 .env 文件中的 DEEPSEEK_API_KEY |
| 向量库构建失败 | 知识库路径错误 | 检查 KNOWLEDGE_BASE 配置 |
| 检索结果为空 | 文档与查询不相关 | 尝试更具体的查询语句 |
| 编码错误 | Windows 终端编码问题 | 确保使用 UTF-8 编码 |

### 8.2 日志输出

系统会在运行过程中输出详细日志，包括：
- 模型加载状态
- 向量库构建/加载信息
- 检索结果统计
- 工作流节点执行状态
