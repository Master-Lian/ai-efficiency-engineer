▶ 无线网络多智能体协同保障系统 | 中兴通讯 | 核心开发 | 2025.04~2025.10

项目描述：针对 5G 网络运维中感知滞后、决策割裂、执行依赖人工的核心痛点，构建基于多智能体协同的自动化运维闭环系统，实现"秒级感知→自动诊断→智能决策→快速执行"的全流程自动化，提升网络运维智能化水平与响应效率。项目依托中兴 5G 网络实践经验构建，可直接适配运营商 5G 基站运维场景。

技术栈：C/C++、Python 3.10、PyTorch、LangChain、LangGraph、rank_bm25、RRF 排名融合、LoRA 微调、DeepSpeed、RLHF、模型蒸馏、vLLM、Redis、Docker、Kubernetes

核心职责与成果：
1. 主导系统架构设计与核心模块开发，搭建感知、决策、执行三智能体协同架构，基于 LangGraph 实现智能体间状态流转与协同，设计 MCP 调度中心统一管理任务分发与状态同步；
2. 采用 PyTorch 框架完成模型开发部署，负责感知智能体 KQI 指标实时计算，复用中兴 VoNR 优化经验，设计 RTP 丢包率统计、连续丢包检测算法，采用环形缓冲区 + 内存池优化性能，确保感知时延 < 1 秒；
3. 搭建决策智能体 RAG 混合检索系统（BM25 关键词检索 + RRF 工业级排名融合），构建运维知识库，调用 Qwen-7B 大模型实现故障诊断推理，通过 LoRA 技术对 Qwen-7B 进行有监督微调，运用 DeepSpeed 分布式训练架构提升训练效率，引入 RLHF 技术结合运维专家反馈优化推理准确性，诊断准确率 > 90%；
4. 设计四层防护机制（诊断验证、安全检查、效果验证、回滚管理），确保"修错了"不会导致更严重的问题，开发执行智能体工具集，助力故障定位时间缩短至 30 秒内；
5. 完成系统容器化与 Kubernetes 部署，设计 HPA 自动扩缩容策略（2-10 副本），配置健康检查与滚动更新机制，确保资源占用符合运维场景需求，推进生产级项目落地。
