## 学习方法建议

### 1. 代码阅读方法

```
第一步：看文件名和注释，理解模块职责
第二步：看类定义和方法签名，理解接口
第三步：看核心方法的实现逻辑
第四步：看调用关系，理解数据流转
第五步：打断点/加日志，实际运行观察
```

### 2. 记忆方法

| 内容 | 方法 |
|------|------|
| 架构图 | 手绘 3 遍以上，形成肌肉记忆 |
| 算法公式 | 理解原理后推导，不要死记 |
| 四层防护 | 编口诀：诊安效回（诊断、安全、效果、回滚） |
| 技术栈 | 分类记忆：语言、框架、模型、部署 |

### 3. 面试准备

```
自我介绍模板（3 分钟）：
1. 项目背景（30 秒）：5G 运维痛点
2. 系统架构（1 分钟）：三智能体 + MCP 调度
3. 技术亮点（1 分钟）：RRF 融合 + 四层防护
4. 成果数据（30 秒）：时延 < 1s、准确率 > 90%
```

### 4. 常见问题应对

| 问题类型 | 应对策略 |
|---------|---------|
| 没听清问题 | "您能再重复一下吗？" |
| 不会回答 | "这个我了解不深，但我的理解是..." |
| 被追问细节 | 回到代码实现，说具体逻辑 |
| 被问对比 | 先说为什么选这个，再说和其他的对比 |

## 能完成吗？

**能，但需要满足以下条件：**

1. **每天投入 7 小时**：不能偷懒，按计划执行
2. **有 Python 基础**：如果完全没接触过 Python，需要额外时间
3. **了解 5G 基本概念**：RSRP、SINR、切换等概念需要先了解
4. **专注代码本身**：不要纠结 LoRA/DeepSpeed/RLHF 等训练细节（简历写了但代码里没有）

### 优先级排序

```
P0（必须掌握）：
  - 三智能体架构
  - LangGraph 状态流转
  - BM25 + RRF 算法
  - 四层防护机制
  - 环形缓冲区 + 内存池

P1（重点掌握）：
  - MCP 调度器
  - 故障检测规则
  - K8s 部署配置
  - 测试用例

P2（了解即可）：
  - LoRA 微调原理（代码里没有）
  - DeepSpeed 训练（代码里没有）
  - RLHF 流程（代码里没有）
  - 模型蒸馏（代码里没有）
```

### 风险提示

简历中提到的 **LoRA 微调、DeepSpeed、RLHF、模型蒸馏** 在代码中没有实现，面试时被问到要诚实说明：

> "这些是我在项目中参与设计和规划的部分，实际代码实现以 RAG 检索和智能体协同为主。我对这些技术有深入理解，能讲清楚原理和应用场景。"
